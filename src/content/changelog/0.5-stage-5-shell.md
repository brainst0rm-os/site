Stage 5 of the implementation plan is in. The shell can now install an app from a local package, persist its registration in the ledger, grant or deny capabilities per-request, and route every host-service call through the IPC envelope + capability check.

This is the foundation the first-party apps (Notes, Database, Files, Graph) build on, and the foundation third-party app developers will use through the SDK in Stage 9.

The architecture decisions behind the install pipeline are in [`apps/03-app-model.md`](https://github.com/brainstorm-app/brainstorm/blob/main/docs/apps/03-app-model.md) and the capability model is in [`security/09-security-and-sandbox.md`](https://github.com/brainstorm-app/brainstorm/blob/main/docs/security/09-security-and-sandbox.md).
